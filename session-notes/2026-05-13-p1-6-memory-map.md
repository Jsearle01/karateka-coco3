# Session: 2026-05-13 — P1.6 memory map

## What landed

docs/memory-map.md establishes CoCo3 memory layout for karateka-coco3.

CPU 64KB layout:
- $0000-$00FF: DP (HAL/engine/system bands per conventions.md §2)
- $0100-$01FF: Stack (256B, conventional 6809 placement)
- $0200-$1FFF: Engine code + data (~7.5KB)
- $2000-$3FFF: HAL code + data (8KB)
- $4000-$5FFF: Content bank window (8KB, bank-switched)
- $6000-$77FF: Working RAM (6KB)
- $7800-$7FFF: Trace ring buffer (256B) + spare
- $8000-$BBFF: Frame buffer A / back buffer ($3C00 = 15,360 B)
- $BC00-$BFFF: MMU alignment pad (1KB, unavoidable $4000-$3C00 gap)
- $C000-$FBFF: Frame buffer B / front buffer ($3C00 = 15,360 B)
- $FC00-$FFFF: Hardware I/O (PIA, GIME registers, CPU vectors)

Physical (128K): upper bank $38-$3F default, lower bank $30-$37
for content streaming via bank window.

Address constants added to src/hal.inc (26 constants covering
frame buffers, trace buffer, GIME registers, mode values).

## Pop-coco3 divergences

Three divergences from pop-coco3 page 12 layout:
- Tighter engine ($0200-$1FFF) + smaller HAL ($2000-$3FFF)
- Stack at $0100-$01FF (conventional) vs pop-coco3 $FE00-$FEFF
- Trace buffer at $7800 (no equivalent in pop-coco3 P1)
Freed $4000-$7FFF used for content window + working RAM + trace.
