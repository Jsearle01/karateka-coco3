# Exhaustive grep-derived ZP map — exec history (2026-07-04)

Produced `docs/project/zp-map.md`: an exhaustive, lifetime-organized audit of
every direct-page reference in the source, so ZP can be chosen + safely reused
without collisions. Documentation audit — **no code change** (prod unchanged).

## Why exhaustive (not partial)
ZP/memory collisions have bitten repeatedly (`sc_pad`/`page_source_blit`;
the restore clobber; present/flip-must-stay-paired; the boot-integration
code-vs-scratch hazard). Root cause each time: **some ZP usage wasn't visible
when another thing allocated.** A partial map reintroduces that invisibility and
gives a false "it's free" signal — worse than none. So the map is built by
grepping EVERY DP reference → mechanically complete + regenerable.

## Method (regenerable — re-grep = re-complete)
1. Extract every `NAME equ $XX` with XX ≤ $FF across engine + HAL + all
   controllers + boot path + all scene drivers/stages (`src/engine/*.s`,
   `src/hal/**/*.s`, `tests/scripted/*.s`).
2. **Split by usage** — a name used as `<NAME` (DP addressing) is a real **ZP
   variable**; a name used only as `#NAME` (immediate) is a **constant** that
   merely has a value ≤ $FF and is NOT zero page (excluded). This split is the
   crux — the naive grep conflated `scene_clk`/`sc_mir` (ZP) with `CLK_*`,
   `PAGE_A_TOKEN`, `EAGLE_SWAP_CLK`, `PR_FLOOR_FILL` (constants).
3. Restrict the collision analysis to the **prod-resident set** (the one
   binary: engine + HAL + the scene-5 arc files); standalone test drivers build
   as isolated binaries (their inline ZP can't co-reside → can't collide).
4. Classify each variable's **lifetime** (durable / scene-scoped / scratch) and
   record aliases (same address, temporally-separated names) with a *why-safe*.

## Coverage
- 44 source files scanned; 75 named `equ ≤ $FF`; after the `<`/`#` split,
  **~60 prod-resident ZP addresses** in use; **15 constant names excluded**.
- 1 raw `<$XX` scratch ref found ($12 = `gfx_initialized`, already a named var).
- **UNRESOLVED: 0** — every DP reference resolved to an owner + lifetime.

## The $52-$54 confirmation (dispatch focus) — STRONGER than "dormant"
The boot-integration verdict called the Akuma `$52-$55` vs
`frame_done/countdown/sync_dc` `$52-$54` overlap *dormant by temporal
separation*. This audit verifies it more strongly: `frame_*` are used ONLY by
`per_frame_main_loop`, and grep confirms **`per_frame_main_loop` is never called
in the prod boot** (boot.s runs the linear scene controller → `jmp scene5_run`;
the "enters per_frame_main_loop" is a stale comment). `frame_sync_dc ($54)` is
never `<`-used in the prod engine. → **zero runtime contention**; Akuma's
$52-$55 is the sole runtime user.

## Latent collisions found (reported, NOT fixed — HS-4)
1. **$52-$54** Akuma vs kernel_per_frame — safe only while `per_frame_main_loop`
   stays unwired; reconcile before enabling a prod per-frame loop.
2. **$51** `sc_pad` vs `page_source_blit` — safe only while `timer_framesync`
   stays out of the active path.
3. **$40/$41** `thr_off` vs `sc_tmpx`/`sc_y` — safe by the discipline that
   `thr_off` is re-set before every use (documented, not a live bug).

## Standing rule (anti-staleness)
Added to the map header: **any dispatch that allocates / relocates / renames a
ZP byte MUST update `docs/project/zp-map.md` (re-grep).** Regenerable by the
method above.
